train_1.1/ · 双路线套件

实现追赶报告 Phase 1 的两条并行路线——Route A 离散化+LoRA（对标 OpenVLA）、Route B 扩散策略（对标 Diffusion Policy）。第 8 周根据实物评估结果二选一。基座模型统一为 Qwen3-VL-2B-Instruct。

Route A · 离散化+LoRA Route B · 扩散策略 Action Chunking K=16 rollout 评估
WHY · 为什么有 train_1.1

从 2022 范式到 2026 前沿

train/ (v1.0) 是 Frozen VLM + MLP + MSE 回归的 2022 范式，存在致命缺陷：MSE 无法处理多模态动作分布、无 Action Chunking、无 rollout 评估、VLM 完全冻结。train_1.1 用现代动作生成方法重建，建立可验证的方法论优势。

⚠基座模型说明：统一使用 Qwen3-VL-2B-Instruct（不是 Qwen2.5 / Qwen2-VL 系列）。Qwen3-VL 自带 Vision Encoder + LLM，不需要额外加载 SigLIP / CLIP。所有代码和文档统一使用 Qwen/Qwen3-VL-2B-Instruct。

ROUTE A · 离散化 + LoRA

对标 OpenVLA

组件实现对应任务
Action Tokenizermodels/action_tokenizer.py — 7D → 256 bins/维 = 1792 tokensP1A.1
离散 Action Headmodels/discrete_vla.py — 输出 (B, K=16, 7, 256) 逐维 logitsP1A.2
LoRA 微调models/lora_utils.py + base.py — Q/K/V/O 加 LoRA (rank=64)P1A.3
Action Chunkingutils/chunking.py — K=16，训练 + 推理时序集成P1A.4
训练脚本train_discrete.py — 逐维交叉熵 lossP1A.5
// LOSS

L = (1/D) Σ_d CE(logits[:,:,:,d], target_tokens[:,:,d])，对所有 K 步求和。

// INFERENCE

argmax/softmax → decode → (B, K, 7) 连续动作 chunk → temporal ensembling 平滑。

ROUTE B · 扩散策略

对标 Diffusion Policy / Octo

组件实现对应任务
DDPM/DDIM 调度器diffusion/scheduler.py — cosine, T=100, DDIM 20 步P1B.2
1D U-Net Denoiserdiffusion/denoiser.py — FiLM 条件注入 + timestep embeddingP1B.1
扩散 VLAmodels/diffusion_vla.py — 冻结 Qwen3-VL + UNet1DP1B.4
Action Chunking扩散在 (B, K=16, 7) 上操作P1B.3
训练脚本train_diffusion.py — epsilon-prediction MSE lossP1B.5
// LOSS

L = MSE(denoiser(x_t, t, cond), ε)，其中 x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε

// INFERENCE

从纯噪声开始，DDIM 20 步去噪 → (B, K, 7) 动作 chunk。

DATA · data.py 占位符

为什么 data.py 留空

data.py 当前是占位符，build_dataloader() 会抛 NotImplementedError。故意不写实现，只冻结接口契约。

// 原因

追赶报告对数据做了几项尚未落地的改动：P0.1 按 episode 80/20 划分、P2.1 多数据集集成（RH20T/DROID/OXE）、P2.2 加权采样。数据 schema 可能从单数据集变成多源混合，action 维度也可能扩展。因此只冻结接口，不写实现。

// 接口契约

build_dataloader(config) 必须返回 (train_loader, val_loader)，每个 batch 必须包含：

Key类型说明
imageList[PIL.Image]交由 Qwen3-VL AutoProcessor 处理
instructionList[str]自然语言任务指令
actionTensor (B, 7)当前帧动作（标准化后）
action_chunkTensor (B, K=16, 7)未来 K 步动作 chunk
raw_actionTensor (B, 7)原始未标准化（评估用）
frame_indexList[int]帧索引
episode_indexList[int]episode 索引

dataset 对象需暴露 action_mean / action_std（np.ndarray, shape (7,)）。填充后 train_discrete.py / train_diffusion.py 无需修改即可运行。

NOTES · 注意事项

关键约定

01
基座模型

统一 Qwen3-VL-2B-Instruct，不要用 Qwen2.5 / Qwen2-VL。

02
data.py 是占位符

训练前必须先填充实现，参照 train/dataset.py。

03
Action 格式

7D EEF delta [dx,dy,dz,ax,ay,az,gripper]，轴角格式。

04
Action Chunking

K=16，chunk[t]=[a_t,...,a_{t+15}]，超界 pad。

05
评估优先

不要只看 loss，必须用 eval/rollout_sim.py 跑成功率。

06
checkpoint

只保存可训练参数（LoRA adapter / denoiser），不保存冻结的 Qwen3-VL。

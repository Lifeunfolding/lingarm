MISSION · 初心与使命

为什么做 LingArm

VLA 领域 95% 的工作跑在 NVIDIA GPU 上，几乎没人考虑边缘部署；而桌面级、低成本、可全链路复现的研究平台，至今仍是空白。LingArm 想填这块空白。

01 / 初心

让每个人都能复现 VLA

不被大算力、大资金门槛挡在门外。¥2000 级机械臂 + 消费级 GPU，就能跑通"看图听指令 → 动手操作"的完整闭环。开源模型权重、训练代码、方案对比——每一层都可审计、可复现。

02 / 使命

低成本闭环 · 全链路打通

从遥操作数据采集（LeRobot）、到 VLA 模型训练（PAI / 本地 RTX）、再到边缘部署（Jetson / K3）。我们不只训练模型，更把"训练—评估—部署"的工程链路做成可复用的脚手架。

03 / 立场

不押注单一算力

Jetson 做边缘实时控制、RTX 做前沿方法训练、K3 RISC-V 做信创国产替代——三线并行，各司其职。既追 OpenVLA / Diffusion Policy 的方法论前沿，也守住自主可控的底线。

04 / 差异化

极低成本边缘 VLA 部署

学术界极少有人在 RISC-V 边缘芯片上做 VLA 实操。这是 LingArm 独特的叙事角度——不是方法最 SOTA，而是第一个证明 VLA 可以在极低成本边缘硬件上做实操，值得 ICRA/CoRL 投稿。

ARCHITECTURE · 方案规划

一体化 VLA vs 多模块方案

当前 VLA 领域没有唯一正确答案。我们对比两条主流路线，并在 baseline 上并行验证——不押注单一方法，用实物 rollout 结果说话。

ROUTE · 一体化 VLA

端到端单模型

代表OpenVLA · Pi0 · RT-2
思路VLM backbone 直接输出 action token，端到端联合训练
LingArmRoute A 离散化 256 bins/维 + LoRA 微调 Q/K/V/O（rank=64）
优点实现简单、社区资源丰富（OpenVLA 全家桶）、VLM 适配机器人领域
风险K3 上 1792 个 softmax 开销可能大，需分组优化

冻结编码器 + 独立动作头

XINCHUANG · 国产替代

SpacemiT K3 RISC-V

进迭时空 · RVA23 · 60 TOPS · 信创
信创场景的国产替代路线。面向对自主可控要求极高的领域，提供可审计技术底座。
  • 全球首颗 RVA23 量产芯片，60 TOPS FP8 原生

// Qwen3-VL-2B-Instruct (FROZEN)
ViT 视觉编码器 + LLM（28 层，hidden=2048）→ last_token_hidden（2048-dim）

// Action Head (TRAINABLE · 513)

⚠ 局限：BridgeData EEF delta only，不能直接用于关节空间机器人，需部署端 IK 转换。单帧输入，无时序上下文。英文指令。

FAQ · 常见问题

你可能想问的

这个模型能直接用在 SO-ARM101 上吗？ +
不能直接用。模型输出的是 EEF delta 7D（与 BridgeData 的 WidowX 一致），需要部署端用 IK 转换为关节角。项目 deploy/eef_delta_to_joints.py 提供了 PyBullet IK 转换。换机械臂只需调校 IK 参数，无需重训。
为什么用 MSE 回归？不是说落后吗？ +
MSE 回归是 v1.0 baseline，确实是 2022 范式，落后约 2 年——它无法处理多模态动作分布、无 Action Chunking、无 rollout 评估。我们保留它作为参照基线，并在 train_1.1/ 实现了离散化+LoRA 与扩散策略两条现代路线，Phase 1 第 8 周二选一。
VRAM 只有 4GB，真的能跑？ +
能。因为 Qwen3-VL-2B 完全冻结（不存梯度），只有 1.3M 的 MLP+Head 可训练。RTX 4060 Laptop 8GB 已实测验证，sdpa attention + bf16 下 VRAM 约 4GB。A10 24GB 是训练环境，T4 16GB 理论也可行。
K3 还在做吗？定位是什么？ +
K3 仍是项目的一部分，但不再是唯一叙事。它是"多线算力"中的国产替代路线，面向信创场景。Jetson 负责实时边缘部署，RTX 负责前沿训练，K3 解决自主可控——三者互补不互斥。Phase 4 会做 K3 的 INT4 量化推理优化。
数据集是真实的吗？ +

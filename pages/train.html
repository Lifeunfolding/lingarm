ARCHITECTURE · 模型架构

Frozen VLA 架构

Qwen3-VL-2B-Instruct 完全冻结，仅训练 MLP Projector 与 Action Head。可训练参数仅 1.3M。

Input: image (448×448) + instruction (text) ↓ [Qwen3-VL-2B-Instruct] ← ❄️ FROZEN (2,127,532,032) Vision Encoder (ViT) + LLM (28L, hidden=2048) ↓ last_token_hidden (2048-dim) [MLP Projector] ← 🔥 TRAINABLE (1,314,824) 2048 → 512 → 512, GELU + Dropout ↓ [Action Head] ← 🔥 TRAINABLE (513) 512 → 7 ↓ Output: action (7D) = [dx, dy, dz, ax, ay, az, gripper] ↑ EEF delta · 轴角格式（不是欧拉角！） 可训练参数合计: 1,315,337 (占总量 0.06%) 总参数量: 2,128,847,369
⚠Action 格式注意：模型输出轴角（axis-angle），不是欧拉角！部署时需注意旋转复合方式。translation_scale 与 rotation_scale 是可学习参数，模型自动适配幅度。

LIMITATIONS · 已知局限

为什么这是 baseline

v1.0 是 2022 范式，存在 4 项致命缺陷。保留它作为参照基线，v1.1 用现代方法重建。

缺陷说明严重性
MSE 连续回归无法处理多模态动作分布，会输出物理上不可能的均值动作🔴 致命
无 Action Chunking单步输出 7D，文档写了 K=16 但代码没实现🔴 致命
无 rollout 评估只有 MSE loss（幻觉指标），不知道模型是否学会操作🔴 致命
VLM 完全冻结2B 模型沦为特征提取器，未适配机器人领域🟡 显著
→这些局限在 train_1.1/ 双路线套件 中被系统性解决：Route A 离散化+LoRA、Route B 扩散策略、Action Chunking K=16、rollout 评估。

DEPLOY · 部署衔接

训练完成后

训练与部署解耦：模型输出 EEF delta，部署端用 IK 转换为关节角。换机械臂只需调校 IK 参数，无需重训。

deploy.py · 推理 + IK 转换
from model import FrozenVLA from eef_delta_to_joints import EEFDeltaConverter # 加载模型（只加载 1.3M 可训练参数） model = FrozenVLA(llm_name="./Qwen3-VL-2B-Instruct", ...) model.load_trainable("epoch_4.pt") # 推理 → EEF delta 7D eef_delta = model(image, instruction) # 部署端 IK 转换 → 关节角 converter = EEFDeltaConverter(ik_backend="pybullet", urdf_path="so101.urdf") target_joints = converter.convert(eef_delta, current_joints)

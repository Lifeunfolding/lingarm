状态：存亡边缘——方法论落后 2 年，竞争对手月更论文。目标：12 个月内从 2022 范式跃迁到 2026 前沿。

LingArm 的差异化优势

01
完整硬件+软件闭环

从机械臂选型、边缘部署到 LeRobot 数据管线，全链路打通的设计思路比大多数只有 Sim 实验的论文强。

02
信创+边缘独特定位

学术界 95% 的 VLA 工作在 NVIDIA GPU 上跑，几乎没人考虑 RISC-V 边缘推理——独特的叙事角度。

03
低成本硬件方案

SO-ARM101 ¥1500-2000，对比 Franka（¥200K+）、WidowX（¥30K），成本是他们的 1/20-1/100。

04
扎实的工程基础

数据管线、训练代码、checkpoint 管理都有专业水准。硬件和工程是优势，但模型方法论是短板。

AUDIT · 状态审计

致命缺失

诚实面对短板——这些是必须立刻解决的问题。

缺失项严重性说明
无任何 rollout 评估🔴 致命不知道模型是否学会了操作，MSE loss 是幻觉指标
MSE 连续回归动作生成🔴 致命无法处理多模态动作分布，会输出物理上不可能的动作
无 Action Chunking🔴 致命模型输出单步 7D，文档里写了 16 步但代码没实现
无验证集划分🔴 致命训练/测试数据混合，无法评估泛化
VLM 完全冻结，LoRA 未用🟡 显著2B 模型沦为特征提取器，没有适配机器人领域
无自建 SO-ARM101 真机数据🟡 显著只用 BridgeData (WidowX)，embodiment gap 大

ARCHITECTURE · 三线架构

ROS2 + LeRobot + 桥接层

"ROS2 做安全执行层，LeRobot 做智能训练层，中间一座桥"的混合架构。

Layer 3: LeRobot VLA 智能层 • 遥操作数据采集（lerobot-record） • VLA 模型训练（SmolVLA / Pi0 / ACT） • 推理输出 Action 序列（7-dim: 6关节 + 夹爪） │ publish 到 /vla_action Layer 2: 桥接层（lerobot_ros2_bridge） ← 核心开发目标 • 接收 /vla_action → 校验 → 安全过滤 • 调用 SoArmCommander → 生成轨迹 → 执行 • 回传 /joint_states + /execution_status │ 关节目标位置 Layer 1: ROS2 执行层（so-arm-moveit-hardware） • SoArmCommander C++ API（关节/笛卡尔/夹爪） • MoveIt 2 运动规划（碰撞检测、轨迹平滑） • so_arm_hardware（Feetech STS3215, 100Hz, 同步写） • so_arm_tasks 任务管理器（Pick&Place、航点序列） │ 串口 UART @ 1M baud Layer 0: SO-ARM101 硬件（Follower 臂） • 6x STS3215 舵机 + 夹爪（7-dim 动作空间） • 5V/12V 电源 + USB-TTL 控制板
A线
ROS2 执行层增强

Leader 遥操作、摄像头集成、安全监控、Python API。里程碑 A-M1~A-M4。

B线
LeRobot 训练层

数据集采集（≥100 eps）、PAI 训练环境、SmolVLA 基线、真实臂评估。里程碑 B-M1~B-M4。

C线
桥接层开发

VLA action → ROS2 执行、安全过滤、多模态输入、一键部署。里程碑 C-M1~C-M4。

ACCEPTANCE · 端到端验收

闭环测试指标

测试场景：操作者说"抓取红色积木放入蓝色盒子"，系统自主执行。

验收项指标验证方法
端到端延迟语言指令 → 机械臂运动 ≤ 2 秒计时（含推理）
任务成功率≥ 50%（基线，后续迭代提升）20 次测试统计
系统稳定性连续运行 30 min 无崩溃长时间运行
资源占用GPU 显存 ≤ 24 GB，CPU ≤ 50%监控工具
可重复性相同配置结果一致（±5%）两次独立训练对比
PRINCIPLE · 核心理念

如果只能记住一句话

❝ 不要用 MSE loss 的高低来判断你的 VLA。用机械臂实际执行任务的成功率。从现在开始，每一天、每一个决策，都问自己：这个改动能不能让下周的 rollout 成功率变得更高？如果不能，先别做。
最重要
优先级排序

评估 > 方法 > 数据 > 模型大小 > 部署。没有评估，所有事都是猜测。实物评估 > Sim 评估 > MSE loss。

最不重要
不要浪费时间

追最新论文的每个 trick、调超参到极致、写漂亮文档、和顶级 lab 比规模——深耕一条路线比每周换方法强。

QUICKSTART · 快速开始

加载并推理

deploy.py — 拉取 + 推理
# 1. 克隆模型仓库 git clone https://huggingface.co/LingArm/LingArm_frozen_preview cd LingArm_frozen_preview # 2. 下载基座模型 huggingface-cli download Qwen/Qwen3-VL-2B-Instruct --local-dir ./Qwen3-VL-2B-Instruct # 3. 一键推理 python deploy.py --checkpoint epoch_4.pt --test-image robot_view.jpg
inference.py — Python 推理
import torch from PIL import Image from model import FrozenVLA # 加载模型 model = FrozenVLA( llm_name="./Qwen3-VL-2B-Instruct", mlp_hidden_dim=512, mlp_depth=2, action_dim=7, attn_implementation="sdpa", # "flash_attention_2" on A100/H100 ) model.load_trainable("epoch_4.pt") model = model.to("cuda").eval() # head 转 bf16（匹配 Qwen3-VL 输出 dtype） model.mlp_projector = model.mlp_projector.to(dtype=torch.bfloat16) model.action_head = model.action_head.to(dtype=torch.bfloat16) # 推理 image = Image.open("robot_view.jpg").convert("RGB") instruction = "pick up the red block" with torch.no_grad(): action = model([image], [instruction]) # action: (1, 7) EEF delta [dx,dy,dz,ax,ay,az,gripper] print(action.float().cpu().numpy())

LIMITATIONS · 已知局限

诚实说明

01
Action space

BridgeData EEF delta only——不能直接用于关节空间机器人，需部署端 IK 转换。

02
Domain

训练于 BridgeData 桌面操作场景，对新环境的零样本泛化能力有限。

03
Single image

仅用当前帧，无视频历史的时序上下文。

04
Language

英文指令 only。

→这些局限正是 train_1.1 双路线套件 要解决的：Action Chunking K=16 引入时序、Route A/B 处理多模态、Phase 2 集成多数据集与自建真机数据。
